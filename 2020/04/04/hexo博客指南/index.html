hexo博客指南

April 04 2020

准备工作

  1. 安装git

  2. 安装Node.js

  3. 安装hexo

    1. 在合适的地方创个文件夹（我的地址D:hexo）

    2. 控制台进入该文件夹后，输入

      npm install hexo -g

    3. 初始化hexo文件夹

      hexo init

    4. 安装所需要的组件

      npm install

  4. 修改hexo部署配置

    1. 在hexo根目录下修改_config.yml文件

    1. 安装hexo-deployer-git插件

      npm install hexo-deployer-git --save

  5. 修改主题

    1. 在网页https://hexo.io/themes/寻找主题

    2. 控制台进入hexo文件夹 输入

      git clone XXXXXXX themes/XXXXX

    3. 修改_config.yml中的theme: xxx改为theme: xxxx，然后重新执行hexo g`来重新生成。

如果出现一些莫名其妙的问题，可以先执行hexo clean来清理一下public的内容，然后再来重新生成和发布。

写博客

  1. 控制台进入hexo文件夹 输入

    hexo new 'xxxxx'

  2. 文件系统hexo/source/_posts内就会生成一个md文件，编辑此文件即可

  3. 生成博客文件

    hexo g

  4. 本地预览

    hexo s

  5. 部署至github

    hexo d

注意事项

插入图片

  1. 安装hexo-asset-image

    npm install https://github.com/CodeFalling/hexo-asset-image --save

  2. 修改_config.yml文件，post_asset_folder:true

  3. 修改完成后，每次new一个文章都会在同目录产生一个同名文件夹用来存放图片，输入![](xxx.jpg)即可访问该文件夹内的图片。

  4. 但是，在本地编辑markdown时，使用该路径会访问该文件夹的上一级，所以推荐使用typora，可以在预览时修改图片根目录。

typora markdown编辑器

之前用的vscode，干别的可以，写md文件太麻烦，于是转用typora。
